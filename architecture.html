Data Flows

Tick Request Flow

→Judge POST /v1/tick with trigger IDs
→Resolve each ID from store
→Loop merchants in store
→Enrich with category digest + peer stats
→Select best trigger per merchant
→Generate category-specific message
→Return actions[] with rationale

Reply Request Flow

→Judge POST /v1/reply with message
→Record turn in history map
→Check STOP → end immediately
→Count prior auto-signals (unified)
→0 signals → probe once (send)
→1+ signals → end gracefully
→Route by role → customer / merchant
